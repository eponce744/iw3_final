Nuestro sistema externo es SAP, para el caso de las clases, eran Cli1 y Cli2, clase de referencia 2025-09-30
Aplicamos un Deserializador para el JSON que viene del sistema externo (SAP), me fije en las clases extras del dia.
Puntos a tener en cuenta:
* La clase JsonUtiles se usa para los deserealizadores

* Dentro de package model vamos a poner todo lo relacionado a la integracion con el SAP (sistema externo)

* Los nombres de las entidades externas los podemos asumir que vienen en Ingles y nosotros las deserealizamos para adaptarlas a nuestro sistema
	entre esas adaptaciones esta pasarlas al Español.
	
* Para el caso de las entidades Orden y Producto creamos una extensión para recibir su código del sistema externo.
	Esto se hace ya que los codigos recibidos asumimos que pueden no ser iguales a los que tenemos en nuestro sistema
	- La extension de Orden es OrdenSap
	- La extension de Producto es ProductoSap 
El resto (chofer, cliente y camión) me parece no son necesarias esas integraciones.

Para el caso de la fecha se usa el metodo auxiliar (o método Helper) parseFechaFlexible para convertir el formato String de Fecha recibido en alguno de los 
formatos que tiene el arreglo "formatos".
Por otro lado usamos el metodo sdf.setTimeZone(java.util.TimeZone.getTimeZone("America/Argentina/Buenos_Aires")) para configurar la zona horaria en sdf.
Finalmente a partir del formato elegido para la variable "sdf" se convierte la cadena fechaStrt a un objeto del tipo java.util.Date
